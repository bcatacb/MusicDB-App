# MusicDB Database Schema

## Overview

SQLite database optimized for music production workflow. Single file (`musicdb.db`) with indexed tables for fast queries on songs, lyrics, audio metadata, and release info.

## Tables

### 1. songs (Core catalog)
```sql
CREATE TABLE songs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  project TEXT,
  version TEXT,
  status TEXT CHECK(status IN ('Idea', 'Writing', 'Recording', 'Mixing', 'Released')) DEFAULT 'Idea',
  bpm INTEGER CHECK(bpm BETWEEN 40 AND 220),
  key TEXT,
  theme TEXT,
  tags TEXT, -- JSON or comma-separated
  created_date DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_edited DATETIME DEFAULT CURRENT_TIMESTAMP,
  distributor TEXT,
  release_date DATE,
  isrc TEXT UNIQUE,
  upc TEXT,
  notes TEXT
);
```

### 2. lyrics (Linked to songs)
```sql
CREATE TABLE lyrics (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id INTEGER REFERENCES songs(id) ON DELETE CASCADE,
  body TEXT,
  complete BOOLEAN DEFAULT FALSE,
  last_edited DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(song_id)
);
```

### 3. audio_files (Audio metadata & paths)
```sql
CREATE TABLE audio_files (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id INTEGER REFERENCES songs(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  file_path TEXT NOT NULL, -- Relative path in /audio/
  size_bytes INTEGER,
  duration_seconds REAL,
  bitrate INTEGER,
  sample_rate INTEGER,
  uploaded_date DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(song_id, name)
);
```

## Indexes for Performance
```sql
CREATE INDEX idx_songs_status ON songs(status);
CREATE INDEX idx_songs_bpm ON songs(bpm);
CREATE INDEX idx_songs_tags ON songs(tags);
CREATE INDEX idx_songs_project ON songs(project);
CREATE INDEX idx_lyrics_song_id ON lyrics(song_id);
```
